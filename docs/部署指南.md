## 二、用户操作（二选一）

### 方式 A：EdgeOne 控制台部署（推荐）

1. **准备代码仓库**：将项目推送到 Git（GitHub / GitCode / Gitee 均可）。
2. **登录 EdgeOne Pages**：访问 [EdgeOne Pages 控制台](https://console.cloud.tencent.com/edgeone/pages)，创建项目。
3. **导入 Git 仓库**：选择「从 Git 导入」，授权并选择 DreamGate 仓库。
4. **框架与构建配置**：
   - 框架预设：Vite（或「其他」）
   - 构建命令：`npm run build`（edgeone.json 已配置，通常自动识别）
   - 输出目录：`dist`
   - Node 版本：默认（若报错补 18）
5. **边缘函数**：`/edge-functions/` 目录会被自动识别为边缘函数，无需额外配置。
6. **环境变量**：**无需任何 Key**（Edge AI 内置零 Key）。可选设置 `VITE_DEGRADE_*` 降级开关。
7. **部署**：点击「部署」，等待构建完成（约 1-2 分钟）。
8. **获取访问 URL**：部署成功后，EdgeOne 会分配 `xxx.edgeone.app` 域名。

### 方式 B：EdgeOne CLI 部署

1. **安装 CLI**：
   ```bash
   npm i -g edgeone
   ```
2. **本地联调（含边缘函数）**：
   ```bash
   edgeone pages dev
   ```
   此命令会启动本地服务并加载 `edge-functions/`，可端到端测试 AI 解析（访问 `/api/llm`）。
3. **部署上线**：
   - 推送代码到 Git 触发自动构建，或
   - 直接部署：`edgeone pages deploy`

## 四、Edge AI 限额与降级

- **限额**：EdgeOne 内置 Edge AI（DeepSeek-V3）每日 50 次调用。
- **超限行为**：超限后边缘函数返 503，前端 `ai.ts` 自动捕获并降级到本地 `ruleParser` 规则解析。**功能不中断**，仅解析精度下降（AI 解析 → 关键词规则解析）。
- **演示建议**：演示前确保当日额度未耗尽；录制 Demo 视频优先在额度充足时进行。

## 五、部署后验证清单

部署成功后，按以下清单逐项验证（访问 EdgeOne 分配的 URL）：

1. **镜之门开场**：访问 `/`（自动跳转 `/#/`），确认 3D 镜面渲染 + 碎镜转场。
2. **画廊种子自动加载**：进入 `/#/gallery`，确认 5 扇门 SVG 图不裂图（首次访问自动加载 5 个示例梦境）。
3. **3D 纹理**：切到 3D 走廊视图，确认 DreamDoor 门贴图非全黑（three.js TextureLoader 加载 SVG 成功）。
4. **梦境房间**：访问 `/#/dream/seed-1`，确认 SVG 图 + 情绪/符号解析展示。
5. **分享链接**：在梦境房间生成分享链接，复制到新隐身窗口打开，确认只读卡片正常渲染（验证 HashRouter + `?d=` 参数）。
6. **情绪报告**：访问 `/#/report`，确认跨梦模式（水 ×3 + 飞翔/坠落对比组）可见。
7. **AI 解析端到端**（关键）：
   - 录入一条新梦境 → 触发 AI 生成。
   - 确认图像生成（Pollinations）+ 解析返回（Edge AI）。
   - 若解析返回规则降级结果（`analysisSource: 'rule'`），查 EdgeOne 控制台日志确认是否额度耗尽或 AI binding 形态问题。
8. **AI binding 探测**（若 AI 解析失败）：访问 `/api/ping`，确认返回 JSON：
   - `aiDefined: true` — AI 全局存在
   - `hasChatCompletions: true` — `AI.chatCompletions` 可用（llm.ts 当前调用方式）
   - 若 `hasChatCompletions: false` 但 `hasChat: true`，需将 `edge-functions/api/llm.ts` 的 `AI.chatCompletions(...)` 改为 `AI.chat.completions.create(...)` 并调整防御性提取分支，重新部署。

## 六、常见问题

### Q1: 部署后访问 `/share/xxx` 返回 404？
**A**：DreamGate 使用 HashRouter，正确 URL 是 `/#/share/xxx`（带 `#`）。BrowserRouter 的旧链接需补 `#`。

### Q2: AI 解析总是走规则降级（`analysisSource: 'rule'`）？
**A**：可能原因：
1. Edge AI 当日 50 次额度耗尽 → 等次日重置，或演示用种子数据（已含 AI 解析结果）。
2. AI binding 未绑定 → 查 `/api/ping` 返回，若 `aiDefined: false`，在 EdgeOne 控制台确认 AI 功能已开启。
3. AI 调用形态不符 → 查 `/api/ping` 的 `hasChatCompletions`/`hasChat`，按第五章 Q8 调整 llm.ts。

### Q3: 构建失败 `npm install ECONNRESET`？
**A**：`.npmrc` 已配置 npmmirror 镜像。若仍失败，在 EdgeOne 控制台环境变量设置 `NPM_CONFIG_REGISTRY=https://registry.npmmirror.com`。

### Q4: 构建失败 Node 版本不兼容？
**A**：`edgeone.json` 默认不指定 nodeVersion。若报 Node 版本错误，编辑 `edgeone.json` 添加 `"nodeVersion": "18"`，重新部署。

### Q5: 3D 走廊门贴图全黑？
**A**：SVG 文件已强制 `width="1024" height="1024"` 属性兼容 three.js TextureLoader。若仍黑，查 DevTools Network 确认 `.svg` 请求 200；若 404 检查 `public/seeds/` 是否在构建产物中。

## 七、Vercel 备用部署（可选）

若 EdgeOne 不可用，可部署到 Vercel 作为备选：

1. 代码已保留 `api/llm.ts`（Vercel Functions 风格，调用 SiliconFlow Qwen2.5-7B）。
2. 在 Vercel 项目设置环境变量 `SILICONFLOW_API_KEY`（从 [硅基流动](https://siliconflow.cn) 获取免费 Key）。
3. Vercel 会自动识别 `api/` 目录为 Serverless Functions。
4. 注意：Vercel 部署用 SiliconFlow 后端（非 Edge AI），需自备 API Key。

**双链接策略**：EdgeOne 为主链接（国内 CDN 快），Vercel 为备链接（国际访问兜底）。
