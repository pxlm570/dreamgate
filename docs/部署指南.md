# DreamGate 部署指南（EdgeOne Pages）

> 本指南指导将 DreamGate 梦境档案馆部署到腾讯云 EdgeOne Pages。
> Task 10 已完成所有部署适配，用户按本指南操作即可上线。

---

## 零、部署环境要求

| 项目 | 要求 | 说明 |
|------|------|------|
| 代码仓库 | GitHub / GitCode / Gitee | 已推送，EdgeOne 支持三家 |
| Node.js | ≥ 18 | EdgeOne 默认提供，无需自备服务器 |
| 构建工具 | Vite 6 | `edgeone.json` 已配置 `npm run build` + `dist` |
| 边缘函数 | EdgeOne 内置 | `edge-functions/` 目录自动识别，无需额外配置 |
| AI 服务 | EdgeOne Edge AI（DeepSeek-V3） | 零 Key，控制台开启即可，50 次/天 |
| 域名 | EdgeOne 分配 `xxx.edgeone.app` | 免费提供 HTTPS，无需自备域名/SSL 证书 |
| 服务器 | 无需自备 | EdgeOne Pages 是 Serverless 托管，全自动 |

**核心优势**：零服务器 + 零 Key + 零域名成本 + 自动 HTTPS。

## 二、用户操作（二选一）

### 方式 A：EdgeOne 控制台部署（推荐）

1. **准备代码仓库**：将项目推送到 Git（GitHub / GitCode / Gitee 均可）。
2. **登录 EdgeOne Pages**：访问 [EdgeOne Pages 控制台](https://console.cloud.tencent.com/edgeone/pages)，创建项目。
3. **导入 Git 仓库**：选择「从 Git 导入」，授权并选择 DreamGate 仓库。
4. **框架与构建配置**：
   - 框架预设：Vite（或「其他」）
   - 构建命令：`npm run build`（edgeone.json 已配置，通常自动识别）
   - 输出目录：`dist`
   - Node 版本：默认（若报错补 18）
5. **边缘函数**：`/edge-functions/` 目录会被自动识别为边缘函数，无需额外配置。
6. **环境变量**：**无需任何 Key**（Edge AI 内置零 Key）。可选设置 `VITE_DEGRADE_*` 降级开关。
7. **部署**：点击「部署」，等待构建完成（约 1-2 分钟）。
8. **获取访问 URL**：部署成功后，EdgeOne 会分配 `xxx.edgeone.app` 域名。

### 方式 B：EdgeOne CLI 部署

1. **安装 CLI**：
   ```bash
   npm i -g edgeone
   ```
2. **本地联调（含边缘函数）**：
   ```bash
   edgeone pages dev
   ```
   此命令会启动本地服务并加载 `edge-functions/`，可端到端测试 AI 解析（访问 `/api/llm`）。
3. **部署上线**：
   - 推送代码到 Git 触发自动构建，或
   - 直接部署：`edgeone pages deploy`

## 四、Edge AI 限额与降级

- **限额**：EdgeOne 内置 Edge AI（DeepSeek-V3）每日 50 次调用。
- **超限行为**：超限后边缘函数返 503，前端 `ai.ts` 自动捕获并降级到本地 `ruleParser` 规则解析。**功能不中断**，仅解析精度下降（AI 解析 → 关键词规则解析）。
- **演示建议**：演示前确保当日额度未耗尽；录制 Demo 视频优先在额度充足时进行。

## 五、部署后验证清单

部署成功后，按以下清单逐项验证（访问 EdgeOne 分配的 URL）：

1. **镜之门开场**：访问 `/`（自动跳转 `/#/`），确认 3D 镜面渲染 + 碎镜转场。
2. **画廊种子自动加载**：进入 `/#/gallery`，确认 5 扇门 SVG 图不裂图（首次访问自动加载 5 个示例梦境）。
3. **3D 纹理**：切到 3D 走廊视图，确认 DreamDoor 门贴图非全黑（three.js TextureLoader 加载 SVG 成功）。
4. **梦境房间**：访问 `/#/dream/seed-1`，确认 SVG 图 + 情绪/符号解析展示。
5. **分享链接**：在梦境房间生成分享链接，复制到新隐身窗口打开，确认只读卡片正常渲染（验证 HashRouter + `?d=` 参数）。
6. **情绪报告**：访问 `/#/report`，确认跨梦模式（水 ×3 + 飞翔/坠落对比组）可见。
7. **AI 解析端到端**（关键）：
   - 录入一条新梦境 → 触发 AI 生成。
   - 确认图像生成（Pollinations）+ 解析返回（Edge AI）。
   - 若解析返回规则降级结果（`analysisSource: 'rule'`），查 EdgeOne 控制台日志确认是否额度耗尽或 AI binding 形态问题。
8. **AI binding 探测**（若 AI 解析失败）：访问 `/api/ping`，确认返回 JSON：
   - `aiDefined: true` — AI 全局存在
   - `hasChatCompletions: true` — `AI.chatCompletions` 可用（llm.ts 当前调用方式）
   - 若 `hasChatCompletions: false` 但 `hasChat: true`，需将 `edge-functions/api/llm.ts` 的 `AI.chatCompletions(...)` 改为 `AI.chat.completions.create(...)` 并调整防御性提取分支，重新部署。

## 六、常见问题

### Q1: 部署后访问 `/share/xxx` 返回 404？
**A**：DreamGate 使用 HashRouter，正确 URL 是 `/#/share/xxx`（带 `#`）。BrowserRouter 的旧链接需补 `#`。

### Q2: AI 解析总是走规则降级（`analysisSource: 'rule'`）？
**A**：可能原因：
1. Edge AI 当日 50 次额度耗尽 → 等次日重置，或演示用种子数据（已含 AI 解析结果）。
2. AI binding 未绑定 → 查 `/api/ping` 返回，若 `aiDefined: false`，在 EdgeOne 控制台确认 AI 功能已开启。
3. AI 调用形态不符 → 查 `/api/ping` 的 `hasChatCompletions`/`hasChat`，按第五章 Q8 调整 llm.ts。

### Q3: 构建失败 `npm install ECONNRESET`？
**A**：`.npmrc` 已配置 npmmirror 镜像。若仍失败，在 EdgeOne 控制台环境变量设置 `NPM_CONFIG_REGISTRY=https://registry.npmmirror.com`。

### Q4: 构建失败 Node 版本不兼容？
**A**：`edgeone.json` 默认不指定 nodeVersion。若报 Node 版本错误，编辑 `edgeone.json` 添加 `"nodeVersion": "18"`，重新部署。

### Q5: 3D 走廊门贴图全黑？
**A**：SVG 文件已强制 `width="1024" height="1024"` 属性兼容 three.js TextureLoader。若仍黑，查 DevTools Network 确认 `.svg` 请求 200；若 404 检查 `public/seeds/` 是否在构建产物中。

## 八、性能优化措施

### 8.1 构建产物优化（已内置）

- **Vite 生产构建**：`npm run build` 自动执行 Tree-shaking + 代码分割 + 压缩
- **sourcemap: 'hidden'**：`vite.config.ts` 配置隐藏 sourcemap（生产环境不暴露源码，但保留排错能力）
- **当前产物体积**：439KB gzip（2670 模块），处于合理范围
- **TRAE 徽章 prodOnly**：开发环境不加载徽章脚本，仅生产环境注入

### 8.2 资源加载优化

| 资源 | 优化措施 | 说明 |
|------|----------|------|
| SVG 种子图 | 20 张共 ~80KB | 矢量图，体积小，无限缩放不失真 |
| 3D 模型 | 程序化生成 | 无外部 .glb/.gltf 文件，零下载 |
| Google Fonts | index.html 预连接 | `preconnect` 提前建立 DNS/TLS |
| Pollinations 图像 | URL 直连 + onError 兜底 | 失败立即降级到种子库，不阻塞 |

### 8.3 运行时性能

- **3D 场景**：移动端自动降级 2.5D（`hardwareConcurrency < 4` 或移动端 UA 检测）
- **滚动驱动**：Lenis + GSAP ScrollTrigger scrub，requestAnimationFrame 调度
- **IndexedDB**：idb 库封装，异步读写不阻塞主线程
- **代码分割**：路由组件可按需 lazy load（当前未启用，遵循 web-dev guideline 全量加载）

### 8.4 CDN 与缓存

- **EdgeOne CDN**：自动全球加速，静态资源边缘缓存
- **HashRouter**：`/#/path` 路由不触发服务器请求，全前端路由
- **SVG 永久缓存**：种子图内容固定，可设置长期缓存头

## 九、应急预案

### 9.1 构建失败

| 故障 | 排查 | 解决 |
|------|------|------|
| `npm install ECONNRESET` | 网络不稳定 | `.npmrc` 已配 npmmirror；仍失败则 EdgeOne 控制台设 `NPM_CONFIG_REGISTRY=https://registry.npmmirror.com` |
| `tsc` 类型错误 | 查看构建日志定位文件 | 本地 `npm run check` 复现，修复后推送 |
| Node 版本不兼容 | 构建日志报版本错误 | `edgeone.json` 加 `"nodeVersion": "18"` |
| 内存溢出 (OOM) | 构建日志报 `JavaScript heap out of memory` | `edgeone.json` 构建命令改为 `node --max-old-space-size=4096 ./node_modules/vite/bin/vite.js build` |

### 9.2 运行时故障

| 故障 | 影响 | 应急 |
|------|------|------|
| Edge AI 50 次额度耗尽 | AI 解析降级为规则解析 | 功能不中断；演示前确认额度；用种子数据演示（已含 AI 结果） |
| Pollinations 图像服务不可用 | 新梦境无 AI 图 | 自动降级到种子库 SVG 占位，不阻塞录入 |
| EdgeOne 平台故障 | 站点不可访问 | 切换 Vercel 备用链接（双链接策略） |
| 3D 渲染崩溃（特定设备） | 画廊不可用 | 自动降级 2.5D；手动切 localStorage 模式开关 |

### 9.3 评委访问保障

- **双链接**：EdgeOne（主）+ Vercel（备），任一故障可切换
- **种子数据**：首次访问自动加载 5 个示例梦境，无需任何 AI 调用即可体验完整流程
- **离线保底**：AI 全失败时仍可浏览已存梦境 + 规则解析新梦境
- **演示前自检**：访问 `/api/ping` 确认 AI binding + 访问 `/#/gallery` 确认种子加载

### 9.4 回滚策略

EdgeOne Pages 每次部署生成版本记录，可在控制台一键回滚到任意历史版本。Git 仓库保留完整 commit 历史，可 `git revert` 任意变更。
