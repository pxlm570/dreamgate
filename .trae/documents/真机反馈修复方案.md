## 任务1:画廊揭幕动画重复播放
**文件**:[src/pages/WorldPage.tsx](file:///d:/webprojds/bytedprojs/bdai-creaproj1/src/pages/WorldPage.tsx)

**根因**:WorldPage 是 "/" 与 "/gallery" 的 layout route;切到 /record、/dream/:id(非子路由)会卸载本组件,回 /gallery 重挂 → 第134行 `crossing` state 重置为 false → 第401行 `{!crossing && <EntranceVeil />}` 重新挂载揭幕幕布。EntranceVeil([src/components/ui/EntranceVeil.tsx](file:///d:/webprojds/bytedprojs/bdai-creaproj1/src/components/ui/EntranceVeil.tsx))无状态、每次挂载必播 1.4s 淡出。

**修复**:
- 新增常量 `GALLERY_ENTERED_KEY = "dg-gallery-entered"`(与已有 `STORAGE_KEY = "dg-gallery-mode"` 同处声明)。
- 新增 `galleryEntered` state,惰性初始化从 `sessionStorage` 读取(本会话已进入过画廊则 true)。
- 第401行改为 `{!crossing && !galleryEntered && <EntranceVeil />}`。
- 新增 effect:`stage === "corridor"` 时延迟 1.6s(> 揭幕 1.4s,避免截断首次深链揭幕)写 sessionStorage 并 `setGalleryEntered(true)`;卸载清定时器。

**场景验证**:
- 首次穿门进画廊:crossing=true → 不播揭幕(零遮罩真连续);1.6s 后标记。✓
- 首次深链 /gallery:galleryEntered=false → 播揭幕盖 3D 初始化;1.6s 后标记。✓
- 重挂(去 /record 回 /gallery):galleryEntered=true(从 sessionStorage)→ 不播揭幕。✓ 直击用户反馈。
- 新会话 cold start:sessionStorage 清空 → 播揭幕。✓
- 安全性:corridor 阶段 `crossing || stage==="corridor"` 各处(第343/357/363/375行)本就为 true,`galleryEntered` 只影响第401行,不污染 crossing 语义。

## 任务2:语音录入(STT)修复 + 反馈
**文件**:[src/components/Record/VoiceCapture.tsx](file:///d:/webprojds/bytedprojs/bdai-creaproj1/src/components/Record/VoiceCapture.tsx)
**使用处**:[src/components/Record/RecordForm.tsx:106](file:///d:/webprojds/bytedprojs/bdai-creaproj1/src/components/Record/RecordForm.tsx#L106)(`onTranscript` append 到梦境原文框)

**根因**:
1. **start/stop 竞态**:同一 recognition 实例 `stop()` 后未等 `onend` 就 `start()` 抛 InvalidStateError,被 catch 后 `setListening(false)` → "停止后再点无反应"。
2. **start 静默失败**:权限拒绝/网络/不支持时 `onerror` 仅 `setListening(false)`,无用户提示 → "点击说话没反应"。
3. **缺反馈**:listening 仅按钮变红,interim 显示在按钮旁 `max-w-[220px] truncate`,用户感知不到"正在录入"。
4. **移动端**:iOS Safari 不支持 Web Speech,但 `webkitSpeechRecognition` 构造函数存在导致 supported 误判。

**修复**:
- **每次 start 新建 recognition 实例**(根治竞态):封装 `startRec()` 返回 `{ok, reason}`;`stopRec()` 调 `rec.stop()` 并立即置 `runningRef.current=false`。`toggle` 按 `runningRef`(真实运行态)而非 `listening` state 决策,避免 state 滞后。原有"挂载时 new 一个常驻实例 + onresult/onerror/onend 绑定"改为按需创建。
- **onerror 分类反馈**:区分 `'not-allowed'`/`'service-not-allowed'`(提示"请允许麦克风权限")、`'no-speech'`(静默重置不扰)、`'network'`/`'audio-capture'`(提示"语音服务暂不可用")。新增 `hint` state + 3s 自动清除定时器。
- **start 失败反馈**:`try/catch rec.start()` 抛错时设 hint。
- **录入反馈 UI**:listening 时在按钮左侧显示脉冲红点 + "正在聆听…" + interim 实时转写大字(允许换行、`max-w-full`,不再 `truncate` 在按钮旁)。用 framer-motion 或 tailwind `animate-pulse`。
- **移动端不支持**:`supported=false` 时不再 `return null` 隐藏,改为渲染 disabled 按钮 + tooltip"当前浏览器不支持语音录入,请用 Chrome/Edge 桌面端";`supported=true` 但 start 失败时同样给 hint(覆盖 iOS Safari 构造函数存在但实际不可用的情况)。
