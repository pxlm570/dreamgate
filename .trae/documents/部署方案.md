# 梦阈 DreamGate — 部署与上线方案

> 版本：1.0　|　日期：2026-07-14　|　适用：初赛提交（07-15 截止）

## 一、部署策略选型

### 方案对比

| 维度 | EdgeOne Pages（主） | Vercel（备） | 自建服务器 |
|---|---|---|---|
| 适配度 | ⭐⭐⭐⭐⭐ 零 Key Edge AI + 静态托管 | ⭐⭐⭐⭐ 静态托管 + Serverless | ⭐⭐ 需自行运维 |
| 成本 | 免费（Pages + 50 次/天 Edge AI） | 免费（Hobby 100GB 带宽） | 服务器费 + 运维 |
| 国内访问 | ⭐⭐⭐⭐⭐ 国内 CDN 加速 | ⭐⭐ 国内访问不稳定 | 取决于机房 |
| 部署复杂度 | 低（Git 导入自动构建） | 低（Git 导入自动构建） | 高 |
| AI 集成 | 内置 Edge AI（零配置） | 需配 SiliconFlow Key | 需自行接入 |

### 结论

**主方案：EdgeOne Pages**（国内访问快、内置 Edge AI 零 Key、免费额度充足）
**备方案：Vercel**（海外访问备链、`vercel.json` 已配好）

双链接策略：Demo 帖同时提供 EdgeOne 主链接 + Vercel 备链接，评委无论网络环境如何都能访问。

## 二、环境配置

### 2.1 构建环境要求

| 项 | 要求 |
|---|---|
| Node.js | ≥ 18（EdgeOne 默认 18，Vercel 默认 20） |
| 包管理器 | npm（项目锁定） |
| 构建命令 | `npm run build`（= `tsc -b && vite build`） |
| 输出目录 | `dist` |
| 路由模式 | HashRouter（`/#/path`，无需 SPA fallback 配置） |

### 2.2 环境变量

| 变量 | 用途 | EdgeOne | Vercel | 本地 .env |
|---|---|---|---|---|
| `VITE_USE_GPT_IMAGE` | 前端开关：走 /api/img | `true` | `true` | `true` |
| `OPENAI_API_KEY` | gpt-image 生图 | 控制台配置 | 控制台配置 | ✅ 已配 |
| `OPENAI_BASE_URL` | API 中转站 | `https://www.bytecatcode.org` | 同左 | ✅ 已配 |
| `OPENAI_IMAGE_MODEL` | 图像模型 | `gpt-image-2` | 同左 | ✅ 已配 |
| `SILICONFLOW_API_KEY` | LLM 解析（仅 Vercel） | 不需要 | 控制台配置 | 不需要 |

> EdgeOne 内置 Edge AI（零 Key），不依赖 `SILICONFLOW_API_KEY`。LLM 解析走 Edge AI binding。
> `.env` 已 gitignore，不进入版本控制。部署平台在控制台配置环境变量。

### 2.3 关键配置文件

- `edgeone.json`：EdgeOne 构建配置（buildCommand / outputDir / 函数目录）
- `vercel.json`：Vercel 部署配置（已配好 SPA rewrite + 函数路由）
- `vite.config.ts`：Vite 构建 + manualChunks 拆包 + dev 中间件
- `.npmrc`：npmmirror 镜像（防 ECONNRESET）

## 三、部署步骤

### 3.1 部署前准备（1h）

| 步骤 | 操作 | 验证标准 |
|---|---|---|
| 1 | 镜像 `api/img.ts` → `edge-functions/api/img.ts`（双边缘函数同步） | `edge-img-sync.test.ts` 通过 |
| 2 | 本地质量门全绿 | check/test/lint/build 全过 |
| 3 | 推送代码到 Git（GitHub/GitCode/Gitee） | `git push` 成功 |
| 4 | 确认 `edgeone.json` 配置正确 | buildCommand=`npm run build`，outputDir=`dist` |

### 3.2 EdgeOne 部署（2-4h，首次排雷）

| 步骤 | 操作 | 预计耗时 |
|---|---|---|
| 1 | 登录 [EdgeOne 控制台](https://console.cloud.tencent.com/edgeone) | 5min |
| 2 | Pages → 创建项目 → 导入 Git 仓库 | 10min |
| 3 | 配置构建：框架=Vite，构建命令=`npm run build`，输出=`dist` | 5min |
| 4 | 配置环境变量（`VITE_USE_GPT_IMAGE=true` + `OPENAI_*`） | 5min |
| 5 | 触发首次构建部署 | 10-30min |
| 6 | 排雷（首次部署常见问题，见下方） | 0-2h |
| 7 | 获取分配域名 `xxx.edgeone.app` | — |

**首次部署排雷预案：**

| 问题 | 原因 | 解决 |
|---|---|---|
| `npm install ECONNRESET` | 国内 npm 源不稳定 | 控制台设 `NPM_CONFIG_REGISTRY=https://registry.npmmirror.com` |
| Node 版本不兼容 | EdgeOne 默认 Node 版本 | `edgeone.json` 加 `"nodeVersion": "18"` |
| 构建内存不足 OOM | vendor-three 体积大 | 构建命令改 `node --max-old-space-size=4096 ./node_modules/vite/bin/vite.js build` |
| Edge AI 404 | binding 未注入 | 确认函数在 `edge-functions/` 目录，`onRequest` 签名正确 |

### 3.3 Vercel 备用部署（30min）

| 步骤 | 操作 |
|---|---|
| 1 | 登录 [Vercel](https://vercel.com) → New Project → 导入同一 Git 仓库 |
| 2 | `vercel.json` 已配好，自动识别 Vite 框架 |
| 3 | 配置环境变量：`SILICONFLOW_API_KEY` + `VITE_USE_GPT_IMAGE=true` + `OPENAI_*` |
| 4 | Deploy → 获取 `xxx.vercel.app` 域名 |

### 3.4 部署后验证（1h）

按以下 8 点逐项验证（主链接 EdgeOne）：

| # | 验证项 | URL | 成功标准 |
|---|---|---|---|
| 1 | 镜之门开场 | `https://xxx.edgeone.app/#/` | 3D 拱门渲染 + 碎镜转场 |
| 2 | 画廊种子加载 | `/#/gallery` | 9 扇门图不裂图 |
| 3 | 3D 纹理 | 切 3D 走廊视图 | 门贴图非全黑 |
| 4 | 梦境房间 | `/#/dream/seed-1` | 图 + 情绪/符号解析展示 |
| 5 | 分享链接 | 生成分享链接 → 隐身窗口打开 | 只读卡片正常渲染 |
| 6 | 情绪报告 | `/#/report` | 跨梦模式可见（水×5 等） |
| 7 | AI 解析端到端 | 录入新梦 → 触发生成 | 图像 + 解析返回（或规则降级） |
| 8 | AI binding 探测 | `/api/ping` | `aiDefined: true` |

> 若 `hasChatCompletions: false` 但 `hasChat: true`：需将 `edge-functions/api/llm.ts` 的 `AI.chatCompletions(...)` 改为 `AI.chat.completions.create(...)` 并重新部署。

## 四、维护策略

### 4.1 日常维护

| 项 | 策略 |
|---|---|
| 代码更新 | Git push → EdgeOne/Vercel 自动触发重新部署 |
| 环境变量 | 控制台修改 → 触发重新部署生效 |
| Edge AI 额度 | 50 次/天免费；超限自动降级规则解析（无需干预） |
| 监控 | EdgeOne 控制台访问统计 + 浏览器 DevTools 人工抽检 |

### 4.2 兜底降级链路

```
gpt-image(/api/img) → Pollinations URL → 种子图(SVG)
LLM(Edge AI) → SiliconFlow(Vercel) → 规则解析(ruleParser)
3D 走廊 → 2.5D 视差 → CSS 拱门降级
3D 分享卡 → 静态模板
```

每一级降级都是自动的，无需人工干预。评委环境即使无 Key、即使 AI 额度耗尽，也能完整体验。

### 4.3 回滚程序

| 场景 | 回滚方式 |
|---|---|
| 构建失败 | EdgeOne 控制台一键回滚到上一版本 |
| 运行时严重 bug | `git revert` → 重新部署 |
| EdgeOne 平台故障 | 切换 Vercel 备用链接 |
| Edge AI 额度耗尽 | 自动降级（无需回滚） |
